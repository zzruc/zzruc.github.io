重点内容1：const 和extern
  • const 对象必须初始化。
  • 多个文件共享const对象，必须使用extern关键字定义该对象。

摘自 Bjarne Stroustrup’s “The C++ Programming Language 4th Edition” (section 15.2 Linkage):

// file1.cpp:
int x = 1;
int b = 1;
extern int c;

// file2.cpp:
int x; // means ‘‘int x = 0;’’
extern double b;
extern int c;

There are three errors here: x is defined twice, b is declared twice with different types, and c is declared twice but not defined.

static int x1 = 1; // internal linkage: not accessible from other translation units

const char x2 = ‘a’; // internal linkage: not accessible from other translation units

The keyword const implies default internal linkage, so if you wanted x2 to have external linkage, you need to precede its definitions with extern:

int x1 = 1; // external linkage: accessible from other translation units
extern const char x2 = ‘a’; // external linkage: accessible from other translation units

重点内容2：常量引用(指向常量的引用)和常量指针，指向常量的指针

常量引用：

  • 只有指向常量的引用，没有常量引用，因为引用不是对象，只是日常表述方便称之为常量引用。

  • 引用的类型必须和它所绑定的对象类型一致的两个例外：1.只要初始值类型可以转化为引用的类型，那么我们就可以将其和reference const绑定，即指向常量的引用可以绑定非常量对象，字面值，表达式。之所以可以这样，可以按中间产生了临时变量来理解。

  • 指向常量的引用仅仅是约束了我们能通过引用做的操作，不能约束我们直接对引用所指向的对象的操作。即我们不能通过指向常量的引用改变其指向的对象，但是我们可以直接对其指向的对象进行更改。

常量指针与指向常量的指针：

  • 常量指针表示该指针是常量，它始终指向同一个地址。它不能保证它所指向的对象不被更改。

  • 指向常量的指针表示其指向的对象是常量，我们不能对其指向的对象进行更改，但是我们可以让该指针指向其他对象。

重点内容3：顶层const和底层const
  • 顶层const是指对象本身是常量，如常量指针；底层const是指该指针所指向的对象是常量（或该引用所绑定的对象是常量）。
  • 顶层const在复制时被忽略，毕竟复制不会改变被复制对象；底层const在复制时必须匹配，我们可以将non-const赋值给const，反之不行。

Exercises Section 2.4.2

Exercise 2.27

Which of the following initializations are legal? Explain why.
(a) int i = -1, &r = 0;
(b) int const p2 = &i2;
(c) const int i = -1, &r = 0;
(d) const int const p3 = &i2;
(e) const int *p1 = &i2;
(f) const int &const r2;
(g) const int i2 = i, &r = i;

  • (a) 引用r的初始化错误，引用只能和对象绑定，不能和字面值或其他表达式绑定。
  • (b) 正确。
  • (c) 正确。
  • (d) 正确。
  • (e) 正确。
  • (f) 错误。没有常量引用。
  • (g) 正确。

Exercise 2.28

Explain the following definitions. Identify any that are illegal.

(a) int i, const cp;
(b) int p1, const p2;
(c) const int ic, &r = ic;
(d) const int const p3;
(e) const int *p;

  • 整型i定义正确，相当于int i=0;；指针cp定义错误，常量指针必须初始化（因为常量对象必须初始化）。
  • 指针p1定义正确；p2定义错误。
  • ic定义错误，常量对象必须初始化；正确，指向常量的引用绑定到ic。
  • p3是指向常量对象的常量指针，必须初始化，定义错误。
  • p是指向常量int类型的指针，定义正确。

Exercise 2.29

Uing the variables in the previous exercise, which of the following assignments are legal? Explain why.
(a) i = ic;
(b) p1 = p3;
(c) p1 = ⁣ &ic
(d) p3 = ⁣ &ic
(e) p2 = p1;
(f) ic = *p3;

  • (a)错误。常量对象ic复制给了非常量对象i。
  • (b)错误。指向常量对象的指针赋值给了普通指针。
  • (c)错误。普通指针指向了常量对象。
  • (d)正确。并且我们不能通过p3改变ic的值。
  • (e)正确。用普通指针去初始化了常量指针（相当与用普通对象初始化常量对象）。
  • (f)错误。ic是常量，不能对其更改。
