基于GitHub Page在Mac上搭建Hexo博客

Posted on 2017-08-10

之前一直在博客园上写博客，写了一个SAS系列，但是使用中发现博客园对Markdown的支持不太友好；另外，一直想有一个用自己名字命名的博客网站的想法。说做就做，用了两个多小时搭建起了这个博客，搭建过程不是很难，但是有很多小细节，为了避免自己遗忘，遂写此文记之。

博客搭建方案的选择

博客搭建方案有很多，我选择的是基于GitHub Page使用Hexo搭建博客。

GitHub Page

要搭建自己的博客，肯定要有自己的虚拟主机，有以下两个选择:

  • 购买自己的服务器空间
  • 使用免费的GitHub Page

第一种方法很Geek,但是除去每个月的服务器租借费用，还需要自己构建后台数据库等等，我仅仅就是写一个博客，记录一下自己的学习笔记，况且最近手上很多项目要做，可不想这么费时费力。所以我选用了GitHub Page,好处在于，只要你拥有GitHub账户，就可以使用GitHub Page,享受300M免费空间。

Hexo

搭建博客的工具有很多，我听到的最多的是WordPress，网上也有推荐jekyll的。三种工具的使用方法如下：

  1. Git+Github+Markdown+hexo （免费）
  2. Git+Github+Markdown+jekyll （免费）
  3. 虚拟主机＋插件＋Wordpress （付费）

三者的优缺点也很明显

Hexo:
可以部署到GitHub上，操作比Jekyll简单，命令少，支持markdown,其生成的是静态博客。缺点是每次在新电脑上使用时，都要重新安装和配置编译环境。这个的确很麻烦，想想你如果换了电脑，又得重新配置一遍博客环境。

Jekyll：可以部署到GitHub上，支持markdown，hexo而言，可以直接在GitHub网页上编辑和发布博客，这个的确很方便，因为它不会把你写博客的环境局限到你当前正在使用的本机上。缺点就是jekyll用的liquid语法对程序员不是很友好，但是功能的确比hexo强大很多，时间充裕的可以选择jekyll。

WordPress：安装方便，功能强大，可扩展性高，插件丰富；缺点就是需要自己购买虚拟主机，且迁移成本高。

域名解析

记录类型的含义

要将域名指向主机服务商提供的IP地址，请选择「A记录」；
要将域名指向主机服务商提供的另一个域名，请选择「CNAME记录」。
建立邮箱：需要设置「MX记录」，根据邮箱服务商提供的MX记录填写。

主机记录的含义

www ：
将域名解析为www.example.com，填写www；
@ ：
将域名解析为example.com（不带www），填写@或者不填写；
mail ：
将域名解析为mail.example.com，通常用于解析邮箱服务器；
* ：
泛解析，所有子域名均被解析到统一地址（除单独设置的子域名解析）；

  • 记录类型 : CNAME
  • 主机记录 : 不填
  • 记录值 : zzruc.github.io. (不要忘记最后的.，zzruc改为你自己的用户名)，点击保存即可。
